# Task artifacts are spec, plan, and task

CW will keep the first-version task artifact set small: `spec.md` for the task contract, `plan.md` for the implementation approach, and `task.md` for the executable checklist covering implementation, verification, and review. Optional `baseline-delta.md` captures candidate project-baseline updates, and optional `resume.md` supports user-triggered continuation; separate context, design, verification, review, and finish Markdown files are not core first-version artifacts.
