# Baseline delta syncs during finish

CW will use an optional task-local `baseline-delta.md` for candidate project baseline updates. The coding agent semantically maintains this delta during task work when stable project facts appear, while kernel helpers only validate format and record workflow events. Active task work may update this delta without changing project baseline files. During finish, CW previews the delta, asks for batch confirmation by default, requires explicit confirmation for high-impact architecture, product, conflict, deletion, or low-confidence changes, and records the sync in the task trace. The first version will not auto-apply baseline deltas without user confirmation. During sync, the coding agent performs semantic edits to the baseline files, while kernel helpers handle validation, trace events, and any available transactional safeguards.
